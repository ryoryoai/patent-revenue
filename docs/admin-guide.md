# 管理画面ガイド

Patent Value Checker 管理画面の操作手順書です。

---

## 1. ログイン

### アクセス URL

```
/admin/index.html
```

このページは `noindex, nofollow` メタタグにより検索エンジンに非公開設定されています。

### 認証方式

Supabase Auth によるメール・パスワード認証を使用しています。

**アクセス制限**: サーバー側の `ADMIN_EMAILS` 環境変数に登録されたメールアドレスのみログイン可能です。正しいパスワードでも、未登録のメールアドレスではアクセスを拒否されます。

### ログイン手順

1. メールアドレスとパスワードを入力する
2. 「ログイン」ボタンを押す
3. 認証成功後、ダッシュボードへ自動遷移する

すでにログイン済みの場合は、ログイン画面に訪問すると自動的にダッシュボードへリダイレクトされます。

### ログアウト

画面右上の「ログアウト」ボタンを押すとセッションが終了し、ログイン画面へ戻ります。

## 2. ダッシュボード

### 統計概要

ページ上部に4つの統計カードが表示されます。

| カード | 内容 |
|--------|------|
| リード合計 | 全期間の登録リード総数 |
| 本日の新規 | 当日に新規登録されたリード数 |
| 未対応の登録 | 対応待ちの詳細登録件数（ステータス: 未対応） |
| 新規問い合わせ | 未対応の問い合わせ件数（ステータス: 新規） |

### タブ構成

ダッシュボードは3つのタブで構成されています。

| タブ | 内容 |
|------|------|
| リード | 診断フォームから登録されたリード一覧 |
| 詳細登録 | 詳細レポートや売却支援への本登録一覧 |
| 問い合わせ | コンタクトフォームからの問い合わせ一覧 |

## 3. リード管理

### リード一覧

「リード」タブに、登録日時・氏名・企業・メール・特許番号・本登録状況・ステータスが表示されます。1回のロードで50件まで表示され、「もっと読み込む」ボタンで追加取得できます。

行をクリックするとリード詳細画面へ遷移します。

### フィルター・検索

| 操作 | 機能 |
|------|------|
| ステータスフィルター | 特定ステータスのリードのみ表示 |
| テキスト検索 | 名前・企業名・メールアドレスで絞り込み（400ms デバウンス） |

フィルターまたは検索条件を変更すると、一覧がリセットされて先頭から再取得されます。

### CSV 出力

「CSV出力」ボタンを押すと、最大500件の特許データを CSV ファイルとしてダウンロードできます。ファイル名は `patents-export-YYYY-MM-DD.csv` の形式です。

### リードのステータス一覧と遷移

```
作成済み → 診断済み → メール送信済み → 詳細レポート開始 → 詳細レポート送信済み → 連絡済み → 成約 → クローズ
```

| ステータス | 説明 |
|-----------|------|
| 作成済み | フォームから登録直後（未診断） |
| 診断済み | 特許診断フォームの送信完了 |
| メール送信済み | 診断結果メールを送信済み |
| 詳細レポート開始 | 詳細レポート申込フォームを開始 |
| 詳細レポート送信済み | 詳細レポート申込フォームを完了 |
| 連絡済み | 担当者がユーザーに連絡した |
| 成約 | 契約・取引が成立した |
| クローズ | 対応終了（見込みなし・辞退等） |

## 4. リード詳細

リード一覧の行をクリックすると、そのリードの詳細ページへ遷移します。

### 表示内容

#### 顧客情報

| 項目 | 内容 |
|------|------|
| 氏名 | 登録者の氏名 |
| 企業 | 企業名 |
| メール | クリックでメーラーが開くリンク形式で表示 |
| ステータス | 現在のステータス（バッジ表示） |
| 流入元 | アクセス元（UTMパラメータ等） |
| 登録日時 | フォーム登録日時 |

#### 特許情報

登録された特許ごとに以下の情報を表示します。

| 項目 | 内容 |
|------|------|
| 特許番号 | 特許番号 |
| タイトル | 特許タイトル |
| 出願人 | 出願者・保有企業名 |
| カテゴリ | 技術カテゴリ |
| 権利状態 | 有効・失効等の権利状況 |
| スコア | 診断スコア（70点以上: 高、40〜69点: 中、39点以下: 低） |
| 価値レンジ | 推定市場価値の範囲 |

#### 詳細登録

本登録が存在する場合、種別・ステータス・担当者・電話・希望価格・サポート方法・メッセージ・登録日時を表示します。

### ステータス変更

「アクション」カードの「ステータス変更」セレクトで新しいステータスを選択し、「更新」ボタンを押します。更新後、詳細情報が自動的に再読み込みされます。

### 管理者メモ

「管理者メモ」テキストエリアに自由記述でメモを入力し、「メモ保存」ボタンで保存します。メモはリード詳細を開くたびに読み込まれます。

## 5. 詳細登録管理

「詳細登録」タブで、詳細レポートや売却支援への本登録一覧を確認できます。

### 表示項目

登録日時・種別・担当者名・希望価格・サポート方法・ステータスが一覧表示されます。

行をクリックすると、対応するリードの詳細ページへ遷移します。

### ステータス一覧

| ステータス | 説明 |
|-----------|------|
| 未対応 | 新規登録、まだ対応していない |

| 連絡済み | ユーザーに連絡した |
| 成約 | 契約・取引が成立した |
| クローズ | 対応終了 |

## 6. 問い合わせ管理

「問い合わせ」タブで、コンタクトフォームからの問い合わせ一覧を確認できます。

### 表示項目

受信日時・氏名・企業名・メールアドレス・問い合わせ種別・ステータスが一覧表示されます。

### ステータス一覧

## 7. 認証の仕組み（技術詳細）

### フロントエンド認証フロー

1. ログイン時に Supabase から JWT アクセストークンを取得
2. 各 API リクエストの `Authorization: Bearer {token}` ヘッダーに付与
3. セッションが切れると自動的にログイン画面へリダイレクト
4. API から 403 が返った場合もログイン画面へリダイレクト

### バックエンド認証

サーバー側（`lib/admin-auth.js`）では以下の順序で認証を検証します。

1. **Supabase Auth JWT**: `Authorization` ヘッダーのトークンを検証し、`ADMIN_EMAILS` に含まれるメールアドレスか確認
2. **API キーフォールバック**: `x-metrics-key` ヘッダーが `METRICS_API_KEY` 環境変数と一致する場合に許可（内部バッチ処理用）

### 環境変数

| 変数名 | 説明 |
|--------|------|
| `ADMIN_EMAILS` | 管理者メールアドレス（カンマ区切りで複数指定可） |
| `METRICS_API_KEY` | API キー認証用シークレット（内部処理向け） |
